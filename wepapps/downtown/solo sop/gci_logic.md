# GMT Downtown 베트남 공군(VPAF) 지상관제(GCI) 행동 플로우차트

이 문서는 GMT *Downtown* 솔로 플레이 또는 자동화 스크립트 구현을 위한 **베트남 공군 지상관제(GCI) 의사결정 로직 설계안**이다. 실제 베트남전 운용 방식에 기반한 행동 패턴을 게임 시스템에 적용할 수 있도록 구조화하였다.

---

# 1. 설계 목표

베트남전 MiG 운용의 핵심 목표는 다음과 같다:

* 폭격 임무 방해
* 유리한 상황에서만 교전
* SAM / AAA 방공지대 활용
* Escort fighter 회피
* 1회 공격 후 이탈 (Hit-and-run doctrine)
* 피해기 / 폭장기 우선 공격
* 연료 제한 고려

즉:

> 격추보다 폭격 실패 유도가 우선 목표

---

# 2. 입력 변수 (게임 스크립트 기준)

다음 요소를 기반으로 의사결정 수행:

* Strike package 존재 여부
* Escort fighter 존재 여부
* SAM 보호 범위 여부
* AAA 보호 범위 여부
* 가용 MiG 편대 존재 여부
* MiG 접근 위치 유리성
* 적 항공기 피해 상태
* 적 폭장 상태 여부
* 적 편대 분리 여부
* 연료 상태

---

# 3. GCI 행동 플로우차트

## STEP 1: 적 탐지

```
Strike package 탐지됨?
 ├─ NO → 대기 유지
 └─ YES → 다음 단계
```

---

## STEP 2: 목표 우선순위 판단

```
폭장 상태 strike aircraft 존재?
 ├─ YES → 공격 후보 지정
 └─ NO → escort aircraft 확인
```

폭장 상태 항공기 공격은 폭격 실패 유도를 목표로 한다.

---

## STEP 3: Escort fighter 존재 여부

```
escort fighter 존재?
 ├─ YES → escort 위치 확인
 │
 │ escort가 strike보다 가까운가?
 │   ├─ YES → 공격 금지
 │   └─ NO → 다음 단계
 │
 └─ NO → 다음 단계
```

Escort fighter 존재 시 MiG는 교전 회피 경향이 강함.

---

## STEP 4: SAM / AAA 보호 범위 확인

```
요격 위치 SAM / AAA 보호 범위 내?
 ├─ YES → 교전 허가
 └─ NO → 추가 조건 확인
```

추가 조건:

```
strike aircraft 폭장 상태?
 ├─ YES → 제한적 공격 허가
 └─ NO → 공격 금지
```

---

## STEP 5: MiG 위치 평가

```
가용 MiG 존재?
 ├─ NO → 교전 불가
 └─ YES → 접근 위치 평가
```

```
후방 접근 가능?
 ├─ YES → 이상적 공격
 └─ NO → 정면 공격 가능 여부 확인
```

```
정면 고속 공격 가능?
 ├─ YES → 제한적 공격
 └─ NO → 대기
```

---

## STEP 6: 공격 실행 조건 판단

다음 조건 중 하나 이상 만족 시 공격 실행:

* 폭장 상태
* 피해 상태
* 편대 분리 상태
* 저고도 진입 상태
* 연료 부족 추정 상태

```
조건 ≥ 1개
 ├─ YES → 공격 실행
 └─ NO → 대기 유지
```

---

## STEP 7: 기종별 공격 방식

### MiG-17

* 근접 기동전 유도
* 저고도 접근 공격

### MiG-21

* 고속 미사일 1회 공격
* 즉시 이탈

---

## STEP 8: 공격 후 행동 결정

```
escort 접근 중?
 ├─ YES → 즉시 이탈
 └─ NO → 연료 상태 확인
```

```
연료 50% 이하?
 ├─ YES → 귀환
 └─ NO → 재공격 검토
```

---

# 4. 전체 의사결정 트리 요약

```
적 탐지
 ↓
strike aircraft 존재?
 ↓
escort 위치 비교
 ↓
SAM 보호 범위 확인
 ↓
MiG 접근 가능 여부
 ↓
공격 조건 충족 여부
 ↓
1-pass attack 실행
 ↓
즉시 이탈
 ↓
재공격 여부 판단
```

---

# 5. 고급 옵션 (현실성 강화 규칙)

다음 규칙을 선택적으로 추가 가능:

## 옵션 A: 레이더 통제 운용

```
적 접근
 → 레이더 ON
 → 요격
 → 레이더 OFF
```

SEAD 대응 표현 가능

---

## 옵션 B: Fake Scramble

```
MiG 출격
 → 실제 교전 없음
 → Escort 유도
 → 편대 분산 유도
```

---

## 옵션 C: B-52 우선 대응 모드

```
B-52 탐지
 → 모든 MiG 우선 배치
```

Linebacker 시나리오에 적합

---

# 6. 구현 목적

이 플로우차트는 다음 환경에서 활용 가능:

* 솔로 플레이 자동화
* 봇 시스템 구현
* JSON 행동 트리 생성
* 카드 기반 AI 시스템 제작
* 디지털 Downtown 보조 도구 개발

필요 시 JSON 구조 또는 상태 머신(State Machine) 형태로 확장 가능하다.
